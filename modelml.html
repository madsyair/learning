Pendahuluan

Mengapa Perbandingan Ini Penting?

Dalam dunia data science, tidak ada satu model yang terbaik untuk semua masalah. Setiap algoritma memiliki kekuatan, kelemahan, dan asumsi yang berbeda.

Panduan ini membandingkan 14 model klasifikasi yang paling banyak digunakan dalam praktik — mulai dari regresi logistik yang sederhana dan interpretable, hingga deep neural network yang powerful namun kompleks. Tujuannya bukan untuk mengatakan model mana yang "terbaik", melainkan membantu Anda memahami kapan, mengapa, dan bagaimana menggunakan setiap model secara tepat.

Model-model ini dikelompokkan menjadi empat keluarga besar: model linear (logistik standar & regularisasi), model probabilistik (Naive Bayes & Bayesian Network), model berbasis jarak & margin (KNN & SVM), dan model ensemble & deep learning (Random Forest, boosting family, dan neural network).

Aspek A & B

Konsep Dasar & Parameter Tuning

Setiap model memiliki "filosofi" berbeda dalam mendekati masalah klasifikasi. Memahami prinsip dasar ini sangat krusial sebelum melangkah ke implementasi.

Secara garis besar, model-model klasifikasi bekerja dengan salah satu dari beberapa pendekatan fundamental. Model linear seperti regresi logistik mencari batas keputusan berupa garis/bidang lurus di ruang fitur — sederhana namun terbatas. Model berbasis pohon (decision tree dan turunannya) membagi ruang fitur secara rekursif menjadi kotak-kotak keputusan. Model berbasis jarak (KNN) mengandalkan prinsip bahwa data yang mirip cenderung berada di kelas yang sama. Model kernel (SVM) memproyeksikan data ke dimensi lebih tinggi agar bisa dipisahkan secara linear. Dan neural network belajar representasi bertingkat melalui lapisan neuron yang saling terhubung.

Parameter tuning adalah seni menyetel "tombol-tombol" pada model agar performanya optimal. Setiap model memiliki hyperparameter berbeda — mulai dari yang sederhana seperti k pada KNN, hingga yang kompleks seperti arsitektur deep neural network (jumlah layer, neuron, learning rate, dropout, batch size, dan puluhan lainnya). Semakin banyak hyperparameter, semakin besar ruang pencarian, dan semakin penting strategi tuning seperti cross-validation, grid search, atau Bayesian optimization.
Aspek C & D

Persiapan Data & Kemampuan Algoritma

Tidak semua model diciptakan sama dalam menangani "kekotoran" data. Beberapa sangat sensitif terhadap pencilan dan missing values, sementara yang lain hampir tidak terpengaruh.

Salah satu keputusan terpenting sebelum membangun model adalah bagaimana menyiapkan data. Pertanyaan-pertanyaan kunci yang harus dijawab: Apakah ada missing values? Apakah ada pencilan (outlier) yang ekstrem? Apakah fitur-fitur memiliki skala yang sangat berbeda? Apakah ada fitur kategorik? Apakah ada multikolinearitas antar fitur?

Secara umum, model berbasis pohon (Decision Tree, Random Forest, GBM, LightGBM, XGBoost, GPBoost) adalah yang paling "santai" — mereka tidak memerlukan scaling, robust terhadap pencilan, dan beberapa bahkan bisa menangani missing values secara native. Di sisi lain, model berbasis jarak dan gradient (KNN, SVM, Neural Network, LSTM, Deep FF) sangat sensitif terhadap skala fitur dan pencilan, sehingga preprocessing menjadi wajib.

Pencilan (Outlier)

Model tree-based hampir kebal terhadap pencilan karena split berdasarkan ranking. Sebaliknya, KNN & regresi logistik sangat sensitif — pencilan menggeser estimasi parameter atau mendistorsi perhitungan jarak.

Missing Values

XGBoost & LightGBM bisa handle missing secara native — mereka belajar arah terbaik (kiri/kanan) untuk data yang hilang. Kebanyakan model lain memerlukan imputasi terlebih dahulu.

Scaling / Normalisasi

Wajib untuk KNN, SVM, Neural Network, LSTM — tanpa scaling, fitur berskala besar mendominasi. Tidak perlu untuk semua model tree-based karena split tidak terpengaruh skala.

Encoding Kategori

Naive Bayes & Bayesian Network secara natural menangani kategori. LightGBM native handle kategori. Kebanyakan model lain memerlukan one-hot encoding atau entity embedding.

Multikolinearitas

Regresi logistik sangat terpengaruh (SE membesar, koefisien tidak stabil). Solusi: regularisasi Ridge/ElasticNet. Tree-based & neural network tidak terpengaruh signifikan.

Data Campuran

LightGBM & Bayesian Network paling natural menangani campuran numerik-kategorik. Neural network bisa menggunakan embedding layer untuk fitur kategorik berdimensi tinggi.

Aspek E, F, G

Kelebihan, Kekurangan, & Kebutuhan Data

Setiap model memiliki sweet spot — konteks di mana ia bersinar dan di mana ia gagal.

Tidak ada "model terbaik" universal. Regresi logistik mungkin terlihat kuno, tapi tetap menjadi pilihan pertama di industri perbankan dan medis karena interpretabilitasnya. XGBoost mendominasi kompetisi Kaggle dan benchmark tabular, tapi memerlukan tuning hati-hati. Deep learning luar biasa untuk data berukuran besar dan kompleks, tapi overkill dan bahkan underperform untuk dataset kecil tabular.

Aturan praktis kebutuhan data: Naive Bayes bisa bekerja dengan hanya 10–50 observasi. Regresi logistik memerlukan 10–20 observasi per variabel prediktor. Random Forest dan XGBoost optimal di 500–1000+. Deep neural network memerlukan ribuan hingga puluhan ribu observasi untuk konvergensi yang baik. Semakin kompleks model, semakin banyak data yang dibutuhkan.

Studi benchmark terkenal oleh Grinsztajn et al. (2022, NeurIPS) menunjukkan bahwa pada data tabular tipikal, model tree-based (Random Forest, XGBoost, GBM) masih mengalahkan deep learning. Deep learning baru unggul pada data sangat besar, data sekuensial, atau data dengan struktur kompleks.

Aspek H, I, J, K

Penanganan Overfitting, Imbalance, Interpretasi & Seleksi Variabel

Overfitting

Overfitting terjadi saat model "menghafal" data training. Tree-based: pruning, limit depth. Boosting: early stopping, learning rate kecil, regularisasi. Neural network: dropout, batch normalization, weight decay. Random Forest paling resistant karena bagging secara alami mengurangi variance.

Imbalanced Data

Strategi universal: class_weight, SMOTE, threshold tuning, stratified CV. XGBoost: scale_pos_weight. LightGBM: is_unbalance. Neural network: focal loss, class weights di loss function. Naive Bayes cukup robust karena prior bisa di-adjust.

Interpretabilitas

White box: Logistik (odds ratio), Decision Tree (visualisasi aturan), Naive Bayes (probabilitas). Black box: Neural network, Deep FF, LSTM. Solusi: SHAP, LIME, TreeSHAP (sangat efisien untuk tree-based), partial dependence plots.

Seleksi Variabel

Built-in: Lasso (koefisien = 0), tree-based (feature importance). Perlu metode eksternal: Logistik standar (stepwise), KNN (PCA/filter), neural network (permutation importance). Boruta algorithm sangat baik untuk Random Forest.

Aspek L

Pemodelan Multiclass

Bagaimana model-model ini menangani klasifikasi dengan lebih dari 2 kelas?

Kabar baiknya, sebagian besar model modern native mendukung multiclass. Decision Tree, Random Forest, Naive Bayes, KNN, dan Bayesian Network secara alami bekerja dengan berapapun jumlah kelas tanpa modifikasi. Neural network dan turunannya menggunakan softmax layer dengan categorical cross-entropy. Boosting family (XGBoost, LightGBM) membangun K tree per iterasi (satu per kelas).

Satu-satunya model yang tidak native multiclass adalah SVM — yang memerlukan strategi dekomposisi: One-vs-Rest (OvR, K model) atau One-vs-One (OvO, K(K-1)/2 model). OvR lebih efisien untuk K besar; OvO lebih akurat tapi jauh lebih lambat.

Tips multiclass: Selalu gunakan macro-F1 (bukan accuracy) sebagai metrik utama jika kelas tidak seimbang. Untuk neural network, label smoothing (0.1) dapat mengurangi overconfidence pada prediksi. XGBoost memerlukan num_class diset eksplisit.
