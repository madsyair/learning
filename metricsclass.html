1. Confusion Matrix (Matriks Konfusi)

Confusion matrix adalah tabel 2×2 (untuk kasus biner) yang merangkum hasil prediksi model terhadap nilai aktual. Ini adalah fondasi dari semua metrik evaluasi klasifikasi.

Confusion
Matrix		 Prediksi Model
Positif (+) Negatif (−)
Aktual Positif (+) TP
True Positive		 FN
False Negative
(Type II Error)
Negatif (−) FP
False Positive
(Type I Error)		 TN
True Negative

Penjelasan Tiap Sel

TRUE POSITIVE (TP)

Aktual Positif, diprediksi Positif. Model benar mendeteksi kasus positif.

Contoh: Pasien sakit → model prediksi "sakit" ✓

TRUE NEGATIVE (TN)

Aktual Negatif, diprediksi Negatif. Model benar mendeteksi kasus negatif.

Contoh: Pasien sehat → model prediksi "sehat" ✓

FALSE POSITIVE (FP) — Error Tipe I

Aktual Negatif, diprediksi Positif. Model salah memberikan alarm.

Contoh: Pasien sehat → model prediksi "sakit" ✗ (False Alarm)

FALSE NEGATIVE (FN) — Error Tipe II

Aktual Positif, diprediksi Negatif. Model gagal mendeteksi kasus positif.

Contoh: Pasien sakit → model prediksi "sehat" ✗ (Missed Detection)

Penting

Total observasi = TP + TN + FP + FN = N
Total aktual positif = TP + FN = P
Total aktual negatif = FP + TN = Nneg
Total prediksi positif = TP + FP
Total prediksi negatif = FN + TN

2. Accuracy (Akurasi)

Akurasi mengukur proporsi prediksi yang benar dari seluruh prediksi.

Rumus
$$\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN} = \frac{\text{Prediksi Benar}}{\text{Total Observasi}}$$

Kapan Digunakan?

✓ Data seimbang (balanced) ✓ Baseline cepat ✓ Semua kelas sama penting

Kapan TIDAK Cocok?

✗ Data imbalanced ✗ Biaya kesalahan tidak sama

Paradoks Akurasi

Pada data imbalanced (misal: 95% negatif, 5% positif), model yang selalu prediksi "negatif" mendapat akurasi 95%, padahal tidak berguna sama sekali! Ini disebut Accuracy Paradox.

3. Precision (Presisi) = PPV (Positive Predictive Value)

Precision mengukur: "Dari semua yang diprediksi positif, berapa yang benar-benar positif?"

Rumus
$$\text{Precision} = \text{PPV} = \frac{TP}{TP + FP}$$

Dihitung dari kolom prediksi positif pada confusion matrix.

Kapan Digunakan?

✓ Biaya False Positive tinggi ✓ Spam detection (jangan sampai email penting masuk spam) ✓ Rekomendasi produk (relevansi penting)

Intuisi

Precision tinggi = model jarang salah alarm. Jika model bilang "positif", kita bisa percaya.

· · ·

4. Recall = Sensitivity = TPR (True Positive Rate)

Recall mengukur: "Dari semua yang benar-benar positif, berapa yang berhasil ditangkap model?"

Rumus
$$\text{Recall} = \text{Sensitivity} = \text{TPR} = \frac{TP}{TP + FN}$$

Dihitung dari baris aktual positif pada confusion matrix.

Kapan Digunakan?

✓ Biaya False Negative tinggi ✓ Deteksi penyakit (jangan sampai pasien sakit lolos) ✓ Fraud detection (jangan sampai fraud tidak terdeteksi) ✓ Deteksi stunting, kemiskinan

Intuisi

Recall tinggi = model jarang melewatkan kasus positif yang sebenarnya. Model sensitif terhadap kelas positif.

Trade-off Precision vs Recall

Precision dan Recall bersifat trade-off: menaikkan satu biasanya menurunkan yang lain. Menurunkan threshold prediksi → lebih banyak prediksi positif → Recall naik, tapi Precision turun (lebih banyak false alarm). Ini yang disebut Precision-Recall Trade-off.

· · ·

5. Specificity = TNR (True Negative Rate)

Specificity mengukur: "Dari semua yang benar-benar negatif, berapa yang berhasil diidentifikasi model sebagai negatif?"

Rumus
$$\text{Specificity} = \text{TNR} = \frac{TN}{TN + FP}$$

Dihitung dari baris aktual negatif pada confusion matrix.

Hubungan dengan Sensitivity

False Positive Rate
$$\text{FPR} = 1 - \text{Specificity} = \frac{FP}{TN + FP}$$

FPR digunakan sebagai sumbu-x pada kurva ROC.

Kapan Digunakan?

✓ Tes medis (pastikan orang sehat tidak salah didiagnosis) ✓ Penting saat False Positive mahal/berbahaya ✓ Berpasangan dengan Sensitivity dalam analisis diagnostik

· · ·

6. NPV (Negative Predictive Value)

NPV mengukur: "Dari semua yang diprediksi negatif, berapa yang benar-benar negatif?"

Rumus
$$\text{NPV} = \frac{TN}{TN + FN}$$

Dihitung dari kolom prediksi negatif pada confusion matrix.

Hubungan PPV & NPV

Empat Perspektif Confusion Matrix
PerspektifMetrikDihitung dari
Baris aktual positifSensitivity/Recall = TP/(TP+FN)→ baris atas
Baris aktual negatifSpecificity = TN/(TN+FP)→ baris bawah
Kolom prediksi positifPrecision/PPV = TP/(TP+FP)→ kolom kiri
Kolom prediksi negatifNPV = TN/(TN+FN)→ kolom kanan

Kapan Digunakan?

✓ Penting untuk menilai keandalan hasil negatif ✓ Skrining medis (jika hasil negatif, seberapa yakin benar-benar negatif?)

7. F1-Score

F1-Score adalah harmonic mean dari Precision dan Recall. Memberikan satu angka yang menyeimbangkan keduanya.

Rumus
$$F_1 = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}} = \frac{2 \, TP}{2 \, TP + FP + FN}$$

Mengapa Harmonic Mean?

Harmonic mean memberikan bobot lebih besar pada nilai yang lebih kecil. Jadi F1 akan rendah jika salah satu dari Precision atau Recall rendah, meskipun yang lain tinggi. Ini berbeda dari arithmetic mean yang bisa "menyembunyikan" nilai rendah.

Perbandingan

Misal Precision = 0.9, Recall = 0.1:
Arithmetic mean = (0.9 + 0.1)/2 = 0.50 (terlihat OK)
Harmonic mean (F1) = 2×0.9×0.1/(0.9+0.1) = 0.18 (terlihat buruk → lebih jujur!)

Kapan Digunakan?

✓ Data imbalanced ✓ Precision & Recall sama penting ✓ Metrik tunggal untuk model comparison ✓ Information retrieval, NLP, klasifikasi teks

8. Fβ-Score (Generalisasi F-Measure)

Fβ-Score adalah generalisasi F1-Score di mana β mengontrol bobot relatif antara Recall dan Precision.

Rumus Umum
$$F_\beta = (1 + \beta^2) \times \frac{\text{Precision} \times \text{Recall}}{(\beta^2 \times \text{Precision}) + \text{Recall}}$$

β > 1 → lebih mementingkan Recall | β < 1 → lebih mementingkan Precision

F2-Score (β = 2)

$$F_2 = 5 \times \frac{P \times R}{4P + R}$$

Recall 4× lebih penting dari Precision.

✓ Deteksi penyakit ✓ Fraud detection ✓ Deteksi stunting

Kasus: Lebih baik ada false alarm daripada melewatkan kasus berbahaya.

F0.5-Score (β = 0.5)

$$F_{0.5} = 1.25 \times \frac{P \times R}{0.25P + R}$$

Precision 4× lebih penting dari Recall.

✓ Spam filter ✓ Rekomendasi konten ✓ Automated trading

Kasus: False alarm sangat mahal; lebih baik melewatkan beberapa positif.

Cara Mengingat β

β = 1: Precision = Recall (seimbang) → F1
β = 2: Recall 4× lebih penting → F2 (ingat: β² = 4)
β = 0.5: Precision 4× lebih penting → F0.5 (ingat: β² = 0.25)
Secara umum, Recall mendapat bobot β² kali lebih besar dari Precision.

9. Cohen's Kappa (κ)

Cohen's Kappa mengukur kesepakatan antara prediksi dan aktual dengan mengoreksi kesepakatan karena kebetulan (chance agreement). Ini mengatasi kelemahan akurasi pada data imbalanced.

Rumus
$$\kappa = \frac{p_o - p_e}{1 - p_e}$$

di mana:
po = observed agreement = Accuracy = (TP+TN)/N
pe = expected agreement by chance

$$p_e = \frac{(\text{Aktual Pos} \times \text{Pred Pos}) + (\text{Aktual Neg} \times \text{Pred Neg})}{N^2}$$

$$p_e = \frac{(TP+FN)(TP+FP) + (FP+TN)(FN+TN)}{N^2}$$

Interpretasi Nilai Kappa

Rentang κInterpretasi
< 0.00Kurang dari chance (lebih buruk dari random)
0.00 – 0.20Slight agreement (sangat lemah)
0.21 – 0.40Fair agreement (cukup)
0.41 – 0.60Moderate agreement (sedang)
0.61 – 0.80Substantial agreement (kuat)
0.81 – 1.00Almost perfect agreement (hampir sempurna)

Sumber: Landis & Koch (1977)

Kapan Digunakan?

✓ Data imbalanced (mengoreksi efek chance) ✓ Membandingkan model yang berbeda pada data yang sama ✓ Multiclass classification ✓ Laporan riset yang memerlukan metrik robust

10. AUC-ROC (Area Under the ROC Curve)

Kurva ROC

ROC (Receiver Operating Characteristic) adalah plot TPR (Sensitivity) vs FPR (1 − Specificity) pada berbagai threshold klasifikasi.

Sumbu Kurva ROC
$$\text{Sumbu-Y: TPR (Recall)} = \frac{TP}{TP+FN} \qquad \text{Sumbu-X: FPR} = \frac{FP}{FP+TN} = 1 - \text{Specificity}$$

AUC (Area Under the Curve)

AUC = luas di bawah kurva ROC. Mengukur kemampuan model membedakan kelas positif dan negatif di seluruh threshold.

Interpretasi AUC

AUCInterpretasi
0.50Sama dengan random guessing (garis diagonal)
0.50 – 0.60Gagal / tidak berguna
0.60 – 0.70Lemah (poor)
0.70 – 0.80Cukup (fair/acceptable)
0.80 – 0.90Baik (good)
0.90 – 1.00Sangat baik / excellent
1.00Klasifikasi sempurna
Penghitungan AUC secara manual (Trapezoid Rule)

Untuk data diskrit (beberapa threshold), AUC dihitung dengan aturan trapesium:

$$\text{AUC} = \sum_{i=1}^{n-1} \frac{(FPR_{i+1} - FPR_i) \times (TPR_{i+1} + TPR_i)}{2}$$

Urutkan titik-titik (FPR, TPR) dari threshold tertinggi ke terendah, lalu jumlahkan luas trapesium.

Interpretasi Probabilistik AUC

AUC = probabilitas bahwa model memberikan skor lebih tinggi pada observasi positif yang dipilih secara acak dibandingkan observasi negatif yang dipilih secara acak.

AUC = 0.85 berarti: jika kita ambil 1 pasien sakit dan 1 pasien sehat secara acak, ada 85% kemungkinan model memberikan skor risiko lebih tinggi pada pasien sakit.

Kapan Digunakan?

✓ Membandingkan model tanpa memilih threshold tertentu ✓ Model menghasilkan probabilitas (bukan hanya label) ✓ Standar dalam riset medis & kredit scoring

Keterbatasan AUC-ROC

Pada data sangat imbalanced, AUC-ROC bisa memberikan gambaran terlalu optimis karena TN mendominasi. Dalam kasus ini, AUC-PR (Area Under Precision-Recall Curve) sering lebih informatif.

12. Contoh Penghitungan Manual Lengkap

Kasus: Prediksi Stunting pada Balita

Sebuah model klasifikasi dibangun untuk mendeteksi stunting pada 200 balita. Hasilnya:

n = 200 Prediksi Total
Stunting (+) Normal (−)
Aktual Stunting (+) TP = 45 FN = 15 60
Normal (−) FP = 20 TN = 120 140
Total 65 135 200

Data dasar: TP=45, TN=120, FP=20, FN=15, N=200, Aktual Positif (P)=60, Aktual Negatif=140

① Accuracy

Accuracy = (TP + TN) / N = (45 + 120) / 200 = 165 / 200 = 0.825 = 82.5%

② Precision (PPV)

Precision = TP / (TP + FP) = 45 / (45 + 20) = 45 / 65 = 0.6923 = 69.23% → Dari 65 balita yang diprediksi stunting, 45 benar-benar stunting.

③ Recall (Sensitivity / TPR)

Recall = TP / (TP + FN) = 45 / (45 + 15) = 45 / 60 = 0.7500 = 75.00% → Dari 60 balita yang benar-benar stunting, model menangkap 45. → 15 balita stunting TIDAK terdeteksi (lolos/missed).

④ Specificity (TNR)

Specificity = TN / (TN + FP) = 120 / (120 + 20) = 120 / 140 = 0.8571 = 85.71% → Dari 140 balita normal, model benar mengidentifikasi 120 sebagai normal. FPR = 1 - Specificity = 1 - 0.8571 = 0.1429 = 14.29%

⑤ NPV (Negative Predictive Value)

NPV = TN / (TN + FN) = 120 / (120 + 15) = 120 / 135 = 0.8889 = 88.89% → Dari 135 yang diprediksi normal, 120 benar-benar normal. → 15 sebenarnya stunting tapi diprediksi normal (bahaya!).

⑥ F1-Score

F1 = 2 × Precision × Recall / (Precision + Recall) = 2 × 0.6923 × 0.7500 / (0.6923 + 0.7500) = 2 × 0.5192 / 1.4423 = 1.0385 / 1.4423 = 0.7200 = 72.00% Cara alternatif: F1 = 2×TP / (2×TP + FP + FN) = 2×45 / (2×45 + 20 + 15) = 90 / 125 = 0.7200 = 72.00% ✓ sama

⑦ F2-Score

F2 = (1 + 2²) × P × R / (2² × P + R) = 5 × 0.6923 × 0.7500 / (4 × 0.6923 + 0.7500) = 5 × 0.5192 / (2.7692 + 0.7500) = 2.5962 / 3.5192 = 0.7377 = 73.77% → F2 > F1 karena Recall (75%) > Precision (69.23%), dan F2 memberi bobot lebih pada Recall.

⑧ F0.5-Score

F0.5 = (1 + 0.5²) × P × R / (0.5² × P + R) = 1.25 × 0.6923 × 0.7500 / (0.25 × 0.6923 + 0.7500) = 1.25 × 0.5192 / (0.1731 + 0.7500) = 0.6490 / 0.9231 = 0.7030 = 70.30% → F0.5 < F1 karena Precision (69.23%) < Recall (75%), dan F0.5 memberi bobot lebih pada Precision.

⑨ Cohen's Kappa (κ)

Langkah 1: Hitung p_o (observed agreement = accuracy) p_o = (TP + TN) / N = (45 + 120) / 200 = 0.825 Langkah 2: Hitung p_e (expected agreement by chance) p_e = (Aktual_Pos × Pred_Pos + Aktual_Neg × Pred_Neg) / N² Aktual Pos = TP + FN = 60 Aktual Neg = FP + TN = 140 Pred Pos = TP + FP = 65 Pred Neg = FN + TN = 135 p_e = (60 × 65 + 140 × 135) / 200² = (3900 + 18900) / 40000 = 22800 / 40000 = 0.5700 Langkah 3: Hitung Kappa κ = (p_o - p_e) / (1 - p_e) = (0.825 - 0.570) / (1 - 0.570) = 0.255 / 0.430 = 0.5930 → Interpretasi: κ = 0.59 → Moderate agreement (Landis & Koch) → Model cukup baik, tapi belum substansial. → Akurasi 82.5% agak "dibesar-besarkan" karena data imbalanced (70% kelas negatif). Kappa mengoreksi ini.

13. Ekstensi ke Multiclass

Untuk klasifikasi dengan K > 2 kelas, confusion matrix menjadi K × K, dan metrik dihitung dengan salah satu strategi:

Strategi Averaging

Macro Average

$$\text{Macro-F1} = \frac{1}{K} \sum_{k=1}^{K} F1_k$$

Hitung metrik per kelas, lalu rata-rata. Setiap kelas sama penting tanpa memperhatikan jumlah observasi.

✓ Semua kelas sama penting

Weighted Average

$$\text{Weighted-F1} = \sum_{k=1}^{K} \frac{n_k}{N} \times F1_k$$

Hitung metrik per kelas, lalu rata-rata berbobot jumlah observasi per kelas.

✓ Data imbalanced

Micro Average

$$\text{Micro-F1} = \frac{2 \sum_k TP_k}{2 \sum_k TP_k + \sum_k FP_k + \sum_k FN_k}$$

Mengumpulkan semua TP, FP, FN global lalu hitung. Micro-Precision = Micro-Recall = Micro-F1 = Accuracy pada multiclass.

Cohen's Kappa untuk Multiclass

Rumus sama persis. po = akurasi keseluruhan, pe dihitung dari marginal total tiap kelas:

$$p_e = \sum_{k=1}^{K} \frac{(\text{Total aktual kelas } k) \times (\text{Total prediksi kelas } k)}{N^2}$$

AUC-ROC untuk Multiclass

Dihitung dengan strategi One-vs-Rest (OvR): untuk tiap kelas k, hitung AUC seolah-olah biner (kelas k vs semua lainnya), lalu rata-rata (macro atau weighted).
